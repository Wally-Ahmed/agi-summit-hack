Part 2 — The Benchmarks
Hermetic by default

Every harness runs stock — exactly as its vendor ships it, nothing extra loaded. No arm gets outside context or added startup cost.

Validated tests

Every task judged only by its own test.py, validated against reference solutions before use.

Integrity tripwire

run.sh refuses to start if benchmark templates differ from git — earned the hard way (see incident below).

Max reasoning everywhere

xhigh (or the harness's maximum) on every arm; fresh workdir per task; sequential arms on one box.

Run 3 — Coordination: Cotal mesh vs direct MCP subagents (the core question)

Same goal, two topologies: mesh (planner → handoff → worker executes in place) vs one orchestrator delegating content to an MCP subagent and transcribing its output.
Cotal mesh
3/3 · 95s
Direct MCP subagent
3/3 · 137s
Mesh wins ~30% at equal quality. Executing in place beats generation-at-a-distance, and the handoff itself costs ≈ nothing: mesh end-to-end ≈ a bare claude -p on the same tasks.

Run 5 — Each harness on its OWN subscription (the real-world mesh question)

The latency story inverts: Claude Code wins ~1.4×. Codex's earlier 2× edge was the backend, not the harness. End-to-end latency is decided by the backend+model pair — exactly the per-task choice a Cotal planner can make.

⚠ The measurement war story (why the tripwire exists)

In the first matrix, Antigravity — run without --add-dir — resolved relative paths against its own workspace and wrote its solutions into the benchmark templates themselves. Every later run copied pre-solved templates into its workdirs: a dead-on-arrival arm "passed" tasks in 10 seconds, and a spurious "native harness scores 3/7" headline nearly shipped. We caught it by autopsying impossible timings, restored templates from git, quarantined the evidence, added a template-integrity tripwire to the runner, and re-ran everything hermetically. The invalidated numbers are preserved in the RESULTS appendix — the methodology story is part of the submission.
